# Commercial Licensing & Contributions

Rusty Demon is distributed under two parallel tracks. Read this file before
contributing or redistributing so everyone's on the same page.

## Source license (this repository)

The source code in this repository is licensed under **AGPL-3.0 with the
Commons Clause** (see [LICENSE](./LICENSE)). In plain English:

- You may read, build, run, study, modify, and share the source for personal
  and educational purposes.
- You **may not sell** the software, or any service whose value derives
  substantially from it, to third parties. That right is reserved to the
  copyright holder (the Rusty Demon maintainers).
- If you distribute a modified version, it must remain under the same
  AGPL + Commons Clause terms and you must make your modifications available.

Because the Commons Clause restricts commercial resale, Rusty Demon is
**source-available**, not FOSS in the strict OSI sense. The distinction
matters if you care about OSI's Open Source Definition; it does not matter
for most practical uses (building, forking for personal use, learning from
the code).

## Commercial distribution (Steam, etc.)

The maintainers reserve the right to distribute compiled builds of Rusty
Demon through commercial channels (e.g. the Steam store) under a separate
proprietary end-user license. Purchasing a commercial build gets you a
packaged, signed, tested binary with support; building from source yourself
remains free and always will.

This dual-track model is the same one used by projects like Aseprite: the
source is public and you can build it yourself, but the convenience of a
packaged commercial build costs money and funds development.

This dual grant is what makes it possible to accept community contributions
while still being able to ship a commercial Steam build. Without it, a
single AGPL-only contribution could block commercial distribution forever.

If you don't agree to these terms, please don't submit contributions.
Opening an issue to report a bug or request a feature does not require
this grant — only contributions that add or modify code, assets, or
documentation do.
